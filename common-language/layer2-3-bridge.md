# Layer-2 ↔ Layer-3 Bridge Template

> This document bridges **Interaction Contract (Layer 2)**  
> with **UI Intent (Layer 3)** — without entering technical territory.

---

## 1. Experience Summary

**User scenario**
- Who the user is
- What situation they’re in

**Core experience promise**
- What the system guarantees from the user’s point of view

---

## 2. Layer 2 — Interaction Contract

> Defines **what must happen** for the experience to be correct.

### Preconditions
- What must already be true before this interaction starts

### User Intent
- What the user is trying to accomplish at this moment

### System Responsibilities
- What the system must do (conceptually, not technically)

### Outcomes
- What state the user ends up in
- What certainty or clarity they gain

---

## 3. Transition: Contract → Intent Mapping

| Interaction Contract (L2) | UI Intent Translation (L3) |
|---------------------------|----------------------------|
| What must be guaranteed   | What the user should feel / understand |
| Required system response  | How that response is perceived |
| State change              | Visual / content acknowledgment |

---

## 4. Layer 3 — UI Intent

> Defines **how the interaction is communicated** to the user.

### Primary UI Intent
- The single most important thing the UI must convey

### Secondary UI Intent
- Reassurance, explanation, or guidance

### Emotional Tone
- Calm / confident / transparent / supportive / neutral

### Information Hierarchy
1. Primary signal
2. Supporting context
3. Optional details

---

## 5. Interaction Expression

**User actions**
- Primary action
- Secondary / fallback action

**System feedback**
- Immediate feedback
- Persistent confirmation (history, status, timestamp, etc.)

**Failure / constraint explanation**
- How limitations are explained without blame

---

## 6. Dependencies

### Experience Dependencies
- Prior flows or knowledge assumed
- First-time vs returning user differences

### Design System Touchpoints
- Existing patterns reused
- Gaps or new patterns emerging

---

## 7. Validation Checklist

- [ ] Interaction contract is unambiguous  
- [ ] UI intent clearly reflects the contract  
- [ ] User can explain what happened in their own words  
- [ ] No hidden state changes  
- [ ] Design decisions are intentional, not decorative
