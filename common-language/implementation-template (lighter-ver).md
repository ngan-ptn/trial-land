# Fast Experiment Prototype Template

> Lightweight template for quick validation and iteration.  
> Optimize for **speed, clarity, and learnings**, not completeness.

---

## 1. Core Concept

> One sentence describing the experience change.

---

## 2. Why This Exists

**User pain / tension**
- What is frustrating, confusing, or risky today?

**Hypothesis**
- We believe that ___ will help users ___ because ___.

---

## 3. Goal

**Primary outcome**
- What should the user be able to do or feel after this flow?

**Success signal**
- How we know this works (behavioral, not metrics-heavy)

---

## 4. Flow (Happy Path Only)

1. User enters from …
2. User sees …
3. User acts …
4. System responds …
5. User reaches outcome

> Skip edge cases unless they block the experiment.

---

## 5. UI Intent (Key Screens Only)

### Screen / Moment
- What the user must immediately understand
- What reassurance or guidance is provided

---

## 6. Interaction Snapshot

- Primary action
- Secondary / escape action
- Feedback shown after action

---

## 7. What We’re Testing

- Assumptions to validate
- Questions we want answered

---

## 8. Notes

- Known limitations
- Follow-ups if validated

---
